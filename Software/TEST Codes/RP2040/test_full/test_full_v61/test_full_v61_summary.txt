Implemented changes
-------------------
1. Added a dedicated IMU lateral compensation controller for `NoWalls`.
   - It activates only when `PidSource::None` is active, meaning both side walls are absent.
   - It injects into the same `lateralCmd` path used by wall-following, not the rotational path.
   - It leaves the existing wall PID untouched when walls are present.
2. Kept the existing IMU yaw controller separate.
   - Yaw hold still contributes only to `rotationCmd`.
   - The new lateral controller does not alter the existing IMU yaw PID math.
3. Implemented the requested 45 degree body-frame transform for IMU planar acceleration.
   - `aForwardBody = 0.7071 * ax + 0.7071 * ay`
   - `aLateralBody = -0.7071 * ax + 0.7071 * ay`
   - Only the signed lateral body-frame component is used for the new controller.
4. Added first-order low-pass filtering on the lateral acceleration signal.
   - The filtered state is updated as `latFilt += alpha * (aLateralBody - latFilt)`.
   - Default filter alpha is `0.12`.
5. Added PD-style lateral control with the requested starting parameters.
   - `imuLatP = 0.1800`
   - `imuLatI = 0.0000`
   - `imuLatD = 0.0300`
   - Output is clamped to `[-1.0, 1.0]`.
6. Added runtime reset handling for the new lateral controller.
   - The lateral runtime is cleared whenever drive PID control is reset.
   - The lateral runtime is also cleared when IMU is globally disabled.
   - The lateral runtime is explicitly reset whenever wall-guided modes are active so stale filtered drift does not leak back in later.
7. Extended the IMU PID submenu with lateral-compensation tuning.
   - Added `Lat P`
   - Added `Lat I`
   - Added `Lat D`
   - Added `Lat LPF`
8. Extended the event-test config menu with the same IMU lateral tuning fields.
   - The new fields are available anywhere the existing IMU yaw PID fields are already configurable for event tests.

Why this change was needed
--------------------------
- In the inherited v56 base, `NoWalls` already zeroed wall PID correction, but there was no dedicated lateral-drift suppression term in the lateral command channel.
- The existing IMU yaw PID could maintain heading, but it does not correct signed sideways body motion.
- That meant open-space travel could keep the right heading while still drifting laterally.

3. New IMU lateral controller core

```cpp
float aForwardBody = IMU_BODY_AXIS_C45 * gImu.ax + IMU_BODY_AXIS_C45 * gImu.ay;
float aLateralBody = -IMU_BODY_AXIS_C45 * gImu.ax + IMU_BODY_AXIS_C45 * gImu.ay;

gImuLatPid.filteredLateralBodyG +=
  gCfg.imuLatFilterAlpha * (aLateralBody - gImuLatPid.filteredLateralBodyG);

float error = -gImuLatPid.filteredLateralBodyG;
float derivative = (error - gImuLatPid.previousError) / dt;
float output = gCfg.imuLatP * error +
               gCfg.imuLatI * gImuLatPid.integral +
               gCfg.imuLatD * derivative;
```
