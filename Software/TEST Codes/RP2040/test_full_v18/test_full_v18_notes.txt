Main PID changes in this version:
- Split PID wall error handling into explicit direction and magnitude instead of sending one combined signed value through the shaping step only.
- Added a shared DirectionMagnitude helper so the PID preview and the real runtime wall-following use the same direction+magnitude path.
- Kept the nonlinear b-curve on magnitude only, then reapplied the direction after shaping.
- Added separate left and right correction output scales in PID settings:
  - L Corr
  - R Corr
- Applied those directional scales to the final PID correction output so weaker left or right steering can be compensated without changing the base wall-error math.

PID menu additions:
- L Corr
- R Corr

PID Test additions:
- Shows raw signed error and explicit direction (L / R / 0)
- Shows raw magnitude and interpreted magnitude separately
- Shows raw PID output and final scaled output separately

Behavior effect:
- The code now treats steering direction and correction magnitude as separate data all the way through the PID preview and wall-follow runtime path.
- Left and right correction strength can be tuned independently if one side of the robot responds weaker than the other.
- Preview and runtime should now match more closely because they share the same raw wall-error helper.
